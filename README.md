# binary_logistic_regression_regularized

# 📐 Линейные модели: реализация логистической регрессии с нуля и с регуляризацией

🧠  Проект выполнен в рамках учебного курса по линейным моделям. Была предоставлена базовая структура кода (функции-заготовки, тесты). Мной самостоятельно реализованы: градиентный спуск, генератор батчей, логистическая регрессия (fit, get_grad, predict), добавлена L1/L2 регуляризация с наследованием классов. Получено качество 0.99 на MNIST.

Этот проект представляет собой **самостоятельную реализацию** ключевых линейных алгоритмов машинного обучения. В нём с нуля написаны:

- **Градиентный спуск** для оптимизации функции двух переменных (демонстрационный пример);
- **Генератор батчей** с перемешиванием для стохастического градиентного спуска (SGD);
- **Логистическая регрессия для бинарной классификации** (базовая версия);
- **Логистическая регрессия с упругой сеткой (ElasticNet)** — одновременная L1 и L2 регуляризация.

Все компоненты написаны на `numpy` без использования готовых моделей из `scikit-learn` (кроме сравнения и тестов). Это позволяет глубже понять, как работают линейные модели изнутри.

## Структура и основные компоненты

### 1️⃣ Градиентный спуск на примере функции $f(x,y)=\sin^2 x + \sin^2 y$

- Аналитически вычислен градиент: $\nabla f = (\sin 2x,\; \sin 2y)$.
- Реализована функция `grad_descent_2d()`, которая сохраняет траекторию движения к минимуму.
- Построена интерактивная 3D‑визуализация с помощью `plotly`, показывающая, как точки спуска сходятся к одной из впадин функции.

### 2️⃣ Генератор батчей для SGD

- Функция `generate_batches(X, y, batch_size)` перемешивает данные в начале каждой эпохи и выдаёт батчи фиксированного размера.
- Последний неполный батч **отбрасывается** — это общепринятая практика для упрощения логики обновления весов.
- Используется `np.random.permutation` для случайного порядка индексов.

### 3️⃣ Обычная логистическая регрессия (класс `MyLogisticRegression`)

Реализованы ключевые методы:

- `fit(X, y, epochs, lr, batch_size)` — обучение с помощью SGD. На каждой итерации:
  - берётся батч из генератора;
  - вычисляются предсказания (сигмоида от линейной комбинации);
  - считается loss (логистическая функция потерь);
  - градиент (по формуле $X^T(p-y)$) обновляет веса.
- `get_grad()` — возвращает сумму градиентов по батчу (без деления на размер батча).
- `predict_proba()` и `predict()` — выдача вероятностей и бинарных меток.
- `__loss()` — кросс-энтропия с клиппингом для численной стабильности.

**Важно**: в веса добавляется **свободный член (bias)** как нулевая компонента вектора $w$.

### 4️⃣ Логистическая регрессия с ElasticNet (класс `MyElasticLogisticRegression`)

Класс **наследуется** от `MyLogisticRegression` и переопределяет:

- `__loss()` — добавляет к логистической функции потерь штрафы:
  $$L = L_{\text{log}} + \alpha \sum_{j=1}^{n}|w_j| + \beta \sum_{j=1}^{n} w_j^2$$
- `get_grad()` — добавляет к основному градиенту производные от регуляризаторов:
  - L1: $\alpha \cdot \text{sign}(w_j)$ (кроме bias!)
  - L2: $2\beta w_j$ (кроме bias!)

Это позволяет управлять как разреженностью весов (L1), так и их затуханием (L2).

## Данные для экспериментов

### Синтетический датасет

С помощью `make_blobs` сгенерированы два облака точек с центрами в $(-2, 0.5)$ и $(3, -0.5)$. Модель успешно разделяет классы, что видно на диаграмме рассеяния и цветовой карте решающей границы.

### Реальный датасет MNIST

- Используется `train.csv` с рукописными цифрами (28×28 пикселей, вытянутые в 784 признака).
- Для бинарной классификации отобраны только классы **0 и 1**.
- Применена **кросс-валидация** (5 фолдов) с предварительной стандартизацией признаков (`StandardScaler`).
- В качестве метрики — **accuracy**.
